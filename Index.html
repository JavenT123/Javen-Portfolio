Network security · home lab

DNS-layer network defense

Deployed a DNS-based filtering and threat-blocking layer across a home network — Pi-hole, Unbound, and UFW working together to filter malicious domains and encrypt upstream resolution.

Pi-holeUnboundUFWFail2ban
View details ↓

Problem

No filtering or privacy at the DNS level — every device on the network sent raw queries straight to the ISP, with full ad and tracker exposure.

What I built

  • Pi-hole as a network-wide DNS sinkhole with custom blocklists
  • Unbound for fully recursive resolution — no upstream provider sees a query
  • UFW locked to only the ports needed, Fail2ban auto-banning failed SSH attempts
  • SSH key-only authentication, password login disabled

What broke / what I learned

First Unbound config returned SERVFAIL on every query — root hints file had gone stale. Re-pulling it from InterNIC fixed resolution immediately. Taught me to actually read DNS error codes instead of guessing at config syntax.
